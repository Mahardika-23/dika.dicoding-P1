Perkenalan

1.1 Tujuan Penulisan Dokumen

Dokumen ini merupakan dokumen Software Requirement Specification (SRS) untuk API pengenalan identitas melalui wajah dengan penerapan teknologi face recognition yang memberikan deskripsi yang rinci mengenai kebutuhan aplikasi yang akan dikembangkan. Tujuan dari penulisan dokumen ini adalah untuk memberikan gambaran yang mendetail dan komprehensif mengenai perangkat lunak kepada pengembang dan stakeholder. Dokumen ini disusun sebagai panduan dalam proses pengembangan dan sebagai sarana evaluasi dari awal hingga akhir pengembangan aplikasi. Dengan adanya SRS ini, diharapkan pengembangan aplikasi dapat dilakukan dengan lebih terstruktur sehingga menghindari keambiguan selama proses pengembangan.


1.2 Audien yang Dituju dan Pembaca yang Disarankan

audiens yang di tuju ialah Aplikasi pihak ketiga yang sedang membutuhkan fungsi pengenalan identitas, fungsi presensi menggunakan wajah, serta fungsi mengubah password hanya menggunakan pengenalan wajah.


1.3 Batasan Produk

Adapun batasan produk dari aplikasi yang dibuat sebagai berikut:

  • Pengguna tidak dapat menggunakan fitur screenshot pada saat mengenal wajah maupun menyetel wajah
  • Pengguna tidak dapat menggunakan aplikasi lain yang memberi akses kamera
  • Pengguna harus terhubung ke internet dengan sinyal minimal 3G
  • Fitur akan dinonaktifkan jika terjadi galat saat masuk yang disebabkan oleh tidak cocoknya wajah terhadap data yang ada. Pengguna harus meminta verifikasi melalui e-mail jika ingin mengaktifkan fitur tersebut kembali.

Deskripsi keseluruhan

2.1 Deskripsi Produk

Aplikasi Pengenalan Identitas Menggunakan Wajah Pihak Ketiga merupakan sebuah aplikasi pihak ketiga yang digunakan sebagai perantara antara pengguna dan aplikasi yang menggunakan aplikasi pihak ketiga kami. Pengguna dapat menyimpan data wajah mereka menggunakan filter kamera pada aplikasi

2.2 Fungsi Produk

Aplikasi Pengenalan Identitas Menggunakan Wajah Pihak Ketiga adalah aplikasi yang menyediakan sistem login yang unik dan mudah bagi pengguna. Dengan memanfaatkan teknologi face recognition, aplikasi ini menggunakan kamera sebagai perantara antara aplikasi dan pengguna dengan ini pengguna dapat mengakses akun dengan mudah dan aman.

2.3 Lingkup Operasi

Adapun lingkungan minimum operasi dari aplikasi ini sebagai berikut :

  1. Windows 8
  2. Browser Opera, Mozilla Firefox, Google Chrome, Brave, dll.
  3. Koneksi Internet minimal 3G

2.4 Batasan Design dan Implementasi

Adapun beberapa hal yang harus diperhatikan dalam mengembangan aplikasi ini yaitu :

  1. Aplikasi ini hanya dapat diakses ketika terdapat jaringan 3G yang menyambung pada smartphone atau desktop. aplikasi ini diwajibkan memiliki koneksi jaringan yang stabil agak proses pengenalan wajah bisa mengirim data secara real time
  2. Fitur kamera pada aplikasi ini hanya dapat digunakan pada perangkat webcam dengan spesifikasi yang sesuai atau memadai

2.5 Deskripsi Produk

Skenario pengguna saat menggunakan aplikasi:

  1. Pengguna membuka aplikasi pihak ketiga, lalu pengguna mengaktifkan fitur face identity. lalu pengguna diminta untuk mengakses kamera untuk menyimpan data wajah ke dalam database.
  2. Setelah mengaktifkan face identity, pengguna bisa mengakses berbagai macam fitur dari aplikasi pihak ketiga seperti mengubah password berdasarkan wajah, melakukan presensi menggunakan wajah, mengambil biodata lengkap pengguna dengan wajah.

Persyaratan Antar Muka

3.1 Antarmuka Perangkat Keras

  1. Personal Komputer juga dapat digunakan sebagai platform interaksi dengan pengguna. Pengguna dapat memasukkan input melalui keyboard fisik, menggunakan mouse untuk navigasi, menggunakan kamera untuk pemrosesan gambar, dan melihat output pada layar monitor.
  2. Keyboard adalah perangkat keras yang digunakan untuk memasukkan data dan perintah oleh pengguna. Dalam aplikasi, pengguna dapat menggunakan keyboard untuk memasukkan informasi seperti pencarian produk, login, atau mengisi formulir.
  3. kamera adalah perangkat seperti smartphone atau webcam pada PC dapat digunakan untuk menangkap gambar atau video lingkungan sekitar pengguna. Kamera ini diperlukan untuk memanfaatkan teknologi Augmented Reality (AR) dalam aplikasi, di mana pengguna dapat melihat objek nyata dengan tambahan elemen digital.
  4. touch screen perangkat keras yang memungkinkan pengguna untuk berinteraksi dengan aplikasi dengan cara menyentuh langsung elemen pada layar. Pengguna dapat menyentuh dan menggesek layar untuk memilih produk, memilih menu, atau melakukan tindakan lainnya dalam aplikasi.
  5. Ethernet adalah perangkat keras yang digunakan untuk menghubungkan perangkat dengan jaringan. Ini memungkinkan sistem untuk berinteraksi dengan server atau layanan online, mengirim dan menerima data melalui jaringan, dan melakukan proses komunikasi lainnya yang diperlukan dalam aplikasi

3.1 Antarmuka Perangkat lunak

  1. Bahasa PHP adalah bahasa pemrograman yang sering digunakan untuk pengembangan aplikasi web. PHP adalah singkatan dari "PHP: Hypertext Preprocessor" yang awalnya merupakan singkatan rekursif yang menunjukkan bahwa PHP digunakan untuk memproses kode yang disisipkan dalam dokumen HTML.
  2. Bootstrap adalah kerangka kerja (framework) front-end yang populer untuk pengembangan aplikasi web. Dikembangkan oleh Twitter, Bootstrap menyediakan kumpulan alat dan gaya yang siap pakai untuk membangun tata letak, desain, dan komponen antarmuka pengguna (UI) yang responsif dan menarik.
  3. MySQL adalah sistem manajemen basis data relasional (RDBMS) yang populer. MySQL digunakan untuk menyimpan dan mengelola data aplikasi, seperti informasi pengguna, produk, pesanan, dan lainnya. Dengan menggunakan MySQL, pengembang dapat membuat dan mengelola tabel-tabel basis data, melakukan kueri data, serta menjaga integritas dan keamanan data aplikasi.
  4. Flutter adalah Flutter adalah framework pengembangan aplikasi mobile open-source yang dikembangkan oleh Google. Dengan menggunakan Flutter, pengembang dapat membuat aplikasi yang dapat berjalan diberbagai platform seperti Android dan iOS. Flutter menggunakan bahasa pemrograman Dart dan menyediakan widget-widget UI yang kaya dan fleksibel untuk membangun tampilan aplikasi yang menarik.
  5. Laravel adalah kerangka kerja (framework) aplikasi web berbasis PHP yang populer. Laravel dirancang untuk memudahkan pengembangan aplikasi web dengan menyediakan berbagai fitur dan alat yang kuat, serta menyajikan sintaks yang ekspresif dan mudah dipahami
